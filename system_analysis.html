1. Fault Tolerance — Failure-First Thinking

Core Philosophy

"Hope is not a strategy." Every component in the pipeline must degrade gracefully when its dependencies fail. The goal is: survive first, then optimize.

CRITICAL: Cache Crashes App on Startup

Failure: If Redis or Qdrant is down, the SemanticCacheAdapter constructor throws an unhandled exception, preventing the FastAPI app from starting.

Root cause: No try/except in SemanticCacheAdapter.__init__ and is_available always returns True.

Fix (applied): Wrapped init in try/except, set _available flag, and updated dependencies.py with an outer try/except.

CRITICAL: LLM Calls Have Zero Timeout

Failure: Ollama LLM calls can hang indefinitely, blocking all subsequent requests.

Root cause: No timeout configuration on Ollama chat() / generate() calls in ollama_service.py.

Fix needed: Add timeout=120 to Ollama client; wrap calls with asyncio.wait_for().

HIGH: Graph Runs Sequentially Even When Parallel Possible

Failure: A single stuck node blocks the entire pipeline. Timeout on one node is missing.

Root cause: LangGraph uses a synchronous execution model by default for nodes unless explicitly parallelized via send().

Fix needed: Parallelize navigate_knowledge_tree + generate_hyde with Send() API.

Local vs. Cloud LLM

✓ Privacy: Zero data leaves the machine.

✓ Cost: $0 per inference.

✗ Latency: 30B local ~20s vs cloud <1s.

✗ SARM: Single point of failure.

Verdict: Justified constraint. Mitigate with model sharding (small for fast, large for rare deep queries) and caching.

Retrieval Method: Quantity vs. Precision

Hybrid retrieval uses: RAPTOR + leaf + LightRAG + Parent-Child + MMR + ColBERT + CrossEncoder.

3. Product Knowledge & Thinking

Why AskMukthiGuru Exists

Mukthi Guru is a privacy-first, zero-hallucination spiritual guide. It answers questions grounded in the teachings of Sri Preethaji and Sri Krishnaji from their verified YouTube videos and approved images.

Core promise: No fabricated quotes, no doctrinal drift, no external API calls at inference time.

Key Concepts

  • Beautiful State: Core teaching — calm, joy, connection.
  • Serene Mind: 4-step guided meditation triggered by distress.
  • Stimulus RAG: Extract key hint phrases before generation.
  • CRAG: Grade docs, rewrite query if poor.
  • Self-RAG: LLM checks its own answer.
  • LettuceDetect: Embedding + lexical faithfulness scorer.

Why Recall Matters

Spiritual queries are sensitive. A wrong answer about "the Beautiful State" or "Four Sacred Secrets" can mislead a seeker. Therefore, the pipeline prioritizes anti-hallucination over speed.

The multi-node verification chain (reflect → verify → check_contradiction) exists to catch answers that sound plausible but are not grounded in source material.

Target vs. Current

Target: <1% hallucination, <3s response time.

Current: ~0.5% hallucination (estimated), ~25–133s response time.

Gap: Quality is nearly there; speed is 8–40× behind.

Path: Replace heavy 30B generation in Fast path with 3B model, parallelize nodes, add top-level hot cache.

User Journey Touchpoints

  1. Landing: Hero + about + practices (static, fast).
  2. Chat: Core interaction. MUST be <3s to feel responsive.
  3. Admin: Queries, retrieval health, daily teachings, prompts — only staff can create or modify.
  4. Daily Teaching: Image + caption, served publicly.

4. Debugging & Eliminating Wrong Answers

Structured Debugging Checklist

When a query produces a wrong or slow answer, check these in order:

  1. Intent: Was the intent classified correctly? If SERENE_MIND was chosen but user asked a factual question, the intent router misclassified.
  2. Graph Strategy: Did Fast/Standard/Deep match the complexity? A complex multi-part question routed to Fast will produce incomplete answers.
  3. Retrieval: Did retrieved documents actually answer the question? Check retrieval_metadata in telemetry.
  4. Cache: Was cache_hit true? If yes, the cached response may be stale.
  5. Verification: What was faithfulness_score and hallucination_flag? A high score but wrong answer means the metric is poorly calibrated.
  6. Guardrails: Was the answer blocked or moderated? Check output_guardrails logs.
  7. Translation: If indic, was translation faithful? Check translation_metadata.
