# AskMukthiGuru Admin Console

A credential-protected `/admin` area providing full observability over the
RAG assistant: usage, retrieval health, RAGAS quality, hallucination detection,
trigger analytics, prompt versioning, evals, safety, cost, alerts, and logs.

## Current phase: UI PREVIEW (mock data)

- All data is generated by `src/admin/lib/seed.ts` (deterministic seed → reproducible screenshots).
- Auth is a localStorage stub. Dev login: **admin / admin**.
- No cloud database is connected yet.

When you're ready to go live, follow `migration-plan.md` step-by-step. The
mock data layer in `src/admin/lib/mockData.ts` was designed so each function
body becomes a `supabase.from(...)` call without changing any UI code.
