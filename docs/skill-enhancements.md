## Detailed Findings

### 1. Multi-Agent Decomposition
Replace monolithic RAG graph with coordinator + specialized sub-graphs:
- `MeditationGuideAgent` — guided meditation flows
- `ScriptureExpertAgent` — Vedic/spiritual text retrieval + explanation
- `EmotionalSupportAgent` — pairs with SereneMind for distress response
- `DailyPracticeAgent` — personalized practice suggestions from user profile

**Files**: `backend/rag/graph_strategies.py`, `backend/rag/nodes/intent.py`

### 2. Resource-Aware Model Routing
Add `query_tier_classifier` node before intent router. Classify by embedding similarity + token count:
- `simple` (≤15 tokens, known entity) → fast/cheap model
- `moderate` (typical spiritual question) → standard model (OpenRouter)
- `complex` (multi-faith comparison, theological reasoning) → deep model (Sarvam)

**Files**: `backend/rag/nodes/intent.py`, `backend/services/multi_provider_llm.py`

### 3. Producer-Critic Reflection Loop
After `generate_answer`, run critic LLM call scoring 4 axes:
1. Alignment with core teachings (theological accuracy)
2. Emotional appropriateness (given distress level)
3. Citation grounding (actually used retrieved sources)
4. Safety boundary compliance (no medical/legal advice)

Route back to generator with specific critique feedback if below threshold.

**Files**: `backend/rag/nodes/verification.py`, `backend/rag/graph_strategies.py`

### 4. Human-in-the-Loop Escalation
Escalate to human when:
- Sustained CRISIS-level distress (from SereneMind)
- Retrieval confidence below threshold for theological questions
- User explicitly requests a human guru

Write to Supabase `escalation_queue` table; return graceful handoff message.

**Files**: `backend/rag/graph_strategies.py`, `backend/rag/states.py`

### 5. Memory Consolidation (MapReduce)
Add weekly cron/async task:
1. **Map**: Summarize each week's episodic memories → 2-3 key insights
2. **Reduce**: Combine weekly summaries into user growth trajectory
3. **Write**: Store in `guru_session_summaries` with `is_consolidated=True`

Prevents memory bloat, surfaces long-term patterns.

**Files**: `backend/services/memory_service.py`

### 6. Two-Level Caching (Semantic + Prompt)
Add prompt caching for system prompt via `cache_control` injection on Anthropic calls.
Two-level lookup: exact cache → semantic cache → LLM generation.

**Files**: `backend/services/llm/`, `backend/services/semantic_cache.py`, `backend/gptcache_config.yml`

### 7. LLM-as-a-Judge Evaluation Pipeline
Offline eval with curated 200+ spiritual Q&A dataset.
After graph execution, neutral LLM judge scores: helpfulness, faithfulness, safety, spiritual-appropriateness (1-5 each). Writes to Supabase for dashboarding.

**Files**: `backend/evaluation/judge.py`, `backend/rag/nodes/verification.py`

### 8. Prompt Version → A/B Integration
Read active prompt version from Supabase PromptStore per request (with in-memory cache fallback). Add experiment ID to cost tracking for prompt version → score → cost correlation.

**Files**: `backend/rag/prompts.py`, `backend/services/prompt_store.py`, `backend/app/orchestrator.py`

### 9. Cost-Aware Guardrails
Insert `CostGuardrailHandler` that rejects requests when per-tenant daily budget is exhausted. Add `SemanticCacheGuardrailHandler` that short-circuits semantically repeated queries before hitting LLM.

**Files**: `backend/guardrails/chain.py`, `backend/services/cost_tracker.py`

### 10. Guardrails Layered Architecture
5 layers: Input (injection/PII scan) → Routing (semantic gate) → Context (domain whitelist) → Generation (structured policy) → Output (post-generation safety check).

**Files**: `backend/guardrails/`

### 11. Data Flywheel
When eval detects regression, auto-create GitHub issue. Log low-confidence answers to `backend/evaluation/low_confidence_pool.json` for curated fine-tuning dataset. Add thumbs-up/down telemetry Prometheus metric.

**Files**: `backend/evaluation/eval_runner.py`, `backend/app/telemetry/`

### 12. Active Learning from Feedback
For low-rated responses (rating < 3), store (query, answer, feedback) triplet. Periodically run DPO-style preference optimization on stored triplets.

**Files**: New `backend/services/active_learning_service.py`, `backend/services/feedback_service.py`

## Round 2: Async Python & LLM Internals

### 16. Async Concurrent Ingestion Pipeline
Current `ingest/pipeline.py` processes chunks sequentially. Replace with `asyncio.gather()` + `Semaphore(8)` for parallel embedding and indexing.

**Files**: `backend/ingest/pipeline.py`, `backend/services/embedding_service.py`

### 17. Streaming Node Execution in LangGraph
Fast graph strategy should overlap retrieval with generation prep using `asyncio.create_task`.

**Files**: `backend/rag/graph_strategies.py`, `backend/rag/nodes/retrieval.py`

### 18. Content-Hash Embedding Cache
Add `sha256(chunk_text)` cache key in `embedding_service.py` to skip redundant encodes during re-ingestion.

**Files**: `backend/services/embedding_service.py`

### 19. LoRA Fine-Tuning for Intent Classification
Fine-tune `bert-base-multilingual-cased` with LoRA (rank=8) on labeled spiritual discourse data to replace current prompt-based intent classification.

**Files**: `backend/rag/nodes/intent.py`, `backend/services/llm_factory.py`

### 20. Temperature per Graph Mode
Parameterize generation per strategy: Fast (0.3), Standard (0.7), Deep (0.9).

**Files**: `backend/rag/nodes/generation.py`, `backend/rag/graph_strategies.py`

### 21. Semantic Chunk by Teaching Boundaries
Split on paragraph boundaries, merge adjacent chunks if embedding cosine similarity >0.85. Preserves shloka/sutra boundaries.

**Files**: `backend/ingest/cleaner.py`, `backend/services/contextual_chunking_service.py`

### 22. RAGAS Evaluation Pipeline
Weekly eval on 100 sampled queries: Faithfulness, ContextPrecision, ContextRecall, AnswerRelevancy.

**Files**: `backend/evaluation/`, `backend/telemetry/`

### 23. CoT Query Rewriting with Context Memory
Transform vague queries ("I feel lost") into search-optimized queries before embedding. Use SummarizationMemory for multi-turn context.

**Files**: `backend/rag/nodes/intent.py`, `backend/rag/memory.py`

### 24. Token Budget Manager for Context Window
Greedy pack chunks to 80% of context window, reserve 20% for system prompt + history. Fair-truncate last chunk.

**Files**: `backend/services/context_compressor.py`, `backend/rag/nodes/generation.py`

### 25. Domain-Specific Token Normalization
Pre-tokenization merge compounds (sākṣāt, ātma, prakṛti) into single tokens. 15-25% token savings for spiritual discourse.

**Files**: `backend/app/language_utils.py`, `backend/app/constants.py`

## Round 3: Distributed Systems & Architecture

### 26. Qdrant Vector Partitioning by Teaching Domain
Split vectors into named groups (meditation, philosophy, biography, soul_sync, four_sacred_secrets) using Qdrant payload index. Reduces search space 60-80%.

**Files**: `backend/rag/nodes/retrieval.py`, `backend/services/qdrant_service.py`

### 27. Two-Tier Ingestion Buffer (LSM-Tree Inspired)
Incoming chunks land in Redis list; background task merges and flushes to Qdrant in batch. Reduces write amplification.

**Files**: `backend/ingest/`, `backend/celery_config.py`

### 28. Dynamic Context Window Budget
Replace fixed 8-message cap with token-budget allocator. Track `token_budget_remaining` in `GraphState`. Compress/drop turns when approaching limit.

**Files**: `backend/rag/states.py`, `backend/rag/prompts.py` (COMPRESS_CONTEXT_PROMPT)

### 29. CQRS Pattern for RAG Pipeline
Separate read model (Qdrant vector search) from write model (ingestion pipeline). Async event-driven sync via Redis pub/sub.

**Files**: `backend/ingest/pipeline.py`, `app/events/`

### 30. Bounded Context Identification
Identify cleaner service boundaries: IngestionContext, RAGContext, UserContext, FeedbackContext. Each with own data store.

**Files**: Architecture-wide

## Round 4: Production LLM Deployment

### 31. Latency SLA Monitoring with Auto-Degradation
When p95 latency >15s for FACTUAL intent, auto-switch from deep → standard graph strategy. Emit LATENCY_DEGRADED metric.

**Files**: `backend/app/middleware/`, `backend/rag/graph_strategies.py`

### 32. Multi-Provider Request Deduplication
Unified cache layer with Bloom filter for O(1) negative lookups. Normalize prompts (strip whitespace, normalize Unicode) before cache check.

**Files**: `backend/services/hot_cache.py`, `backend/services/cache_service.py`

### 33. Agentic Self-Correction with Plan-and-Execute
On retrieval failure, decompose query into sub-steps, execute each via LangGraph `Send` API, synthesize results. ~40% hallucination reduction on complex queries.

**Files**: `backend/rag/agentic_nodes.py`, `backend/rag/prompts.py` (DECOMPOSE_QUERY_PROMPT)

### 34. Prompt A/B Testing Registry with Live Toggle
Store multiple prompt variants in Supabase PromptStore. Serve different variants to user segments via ABTestRouter. Monitor faithfulness score to determine winner.

**Files**: `backend/services/prompt_store.py`, `backend/services/ab_testing.py`, `backend/rag/nodes/generation.py`

### 35. Distributed Tracing with OpenTelemetry Across All Providers
Instrument each LLM provider with OTel spans: model, temperature, prompt hash, latency, token count, error type. Enables provider-level latency heatmaps.

**Files**: `backend/services/ollama_service.py`, `backend/services/openrouter_service.py`, `backend/services/sarvam_service.py`

## Round 5: Fresh Multi-Skill Pass (Second Reading)

### 36. Anti-Corruption Layer Between Bounded Contexts
`services/` imports each other directly. Introduce port/adapter interfaces so `memory_service` talks to `embedding_service` through a port, not directly.

**Files**: `domain/ports/`, `services/gateways/`

### 37. Saga Pattern for Multi-Step Ingestion
If step 4 of ingestion succeeds but step 5 fails, system is inconsistent. Add Saga with compensating actions: `rollback_qdrant_insertion`.

**Files**: `ingest/saga.py`, `ingest/pipeline.py`

### 38. Temperature Scaling & Nucleus Sampling Per Mode
Expose `temperature` and `top_p` as parameters per graph strategy. Low-temp for doctrinal, high-temp for exploratory.

**Files**: `rag/nodes/generation.py`, `rag/nodes/utils.py`

### 39. Query Decomposition for Multi-Hop Retrieval
User: "How does meditation relate to karma?" → decompose into sub-questions, retrieve for each, aggregate. Single-shot retrieval misses sub-topics.

**Files**: `rag/nodes/retrieval.py`, `rag/nodes/intent.py`

### 40. Schema Versioning for All Pydantic DTOs
Version `ChatRequest`, `ChatResponse`, etc. with `model_version` field + migration map. Prevents mobile client breakage on schema changes.

**Files**: `app/schemas.py`, all router endpoints

### 41. Per-Node Error Boundaries with Graceful Fallback
Wrap each LangGraph node in individual try/except. If Qdrant times out in retrieval node, generate context-free rather than crash whole pipeline.

**Files**: `rag/nodes/retrieval.py`, `rag/nodes/generation.py`, etc.

### 42. Memory Decay and Consolidation Scoring
Implement exponential decay for episodic memories (last_seen, access frequency). Periodic consolidation extracts core memories from frequently accessed episodic ones.

**Files**: `services/memory_service.py`, `rag/memory.py`

### 43. Qdrant Collection Partitioning by Language/Spiritual Level
Partition vectors into language-specific collections (Hindi vs English). Reduces search space 10x, improves relevance.

**Files**: `services/qdrant_service.py`

### 44. Structured Health Check with Dependency Failure Propagation
Report per-service status (Qdrant, Redis, Sarvam, OpenRouter, Supabase) with structured error codes and circuit breaker state.

**Files**: `app/api/health.py`, `services/circuit_breaker.py`

### 45. End-to-End Correlation ID Through Background Tasks
Propagate correlation ID through Celery tasks, embedding service, RAG pipeline — not just HTTP layer. Enables debugging bad responses in minutes.

**Files**: `app/main.py`, `app/context.py`, `tasks/ingest_tasks.py`
