# Glossary

Here is an alphabetical glossary of key terms based on the provided chapter titles:

**Batch Processing** — A method of processing multiple tasks or operations in sequence, often used in computer systems to improve efficiency. (Ch 5, Ch 10)

**Clock Synchronization** — The process of ensuring that clocks across a distributed system are aligned and functioning correctly. (Ch 8, Ch 11)

**Data Flow Analysis** — A technique used to analyze the flow of data through a system, often used in compiler optimization and parallel computing. (Ch 5, Ch 9)

**Parallel Computing** — The use of multiple processors or computing units working together on a single task, allowing for increased speed and efficiency. (Ch 4, Ch 10)

**Process Mapping** — A method to visualize and analyze the relationships between different processes in a system, often used in system design. (Ch 6, Ch 7)

**Real-time Processing** — Processing data or tasks within a specified time constraint, requiring deterministic performance. (Ch 2, Ch 10)

**Resource Allocation** — The process of assigning and managing resources such as CPU time, memory, and storage to different processes or tasks. (Ch 3, Ch 6)

**Scaled System** — A system designed to handle a large number of users or tasks while maintaining performance and reliability. (Ch 10)

**Synchronized Clocks** — Clocks that are synchronized with each other to ensure accurate timing in distributed systems. (Ch 8, Ch 11)

**System Design** — The process of designing computer systems, focusing on architecture, functionality, and performance optimization. (Ch 4, Ch 7)

**Timing Constraints** — Specific time limits or requirements that a system must meet to function correctly or efficiently. (Ch 5, Ch 9)
