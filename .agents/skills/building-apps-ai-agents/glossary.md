# Glossary

Here is an alphabetical glossary of key terms based on the provided chapter titles:

**Data collection** — Gathering information or data relevant to a project or study (Ch 1).  
**Adversarial examples** — Inputs designed to cause machine learning models to make mistakes, often used to test robustness (Ch 4, Ch 16).  
**Current state of the agent** — The progress and status of an AI agent in its development cycle (Ch 2).  
**Untested agent** — An untrusted or untested AI agent due to lack of rigorous evaluation (Ch 3 [referenced as Ch 9]).  
**Model performance** — The effectiveness of a machine learning model, often measured through testing and evaluation (Ch 13).  
**Large language models** — Complex AI systems capable of understanding and generating human language (Ch 14).  
**Model architecture** — The structure or design of a machine learning model (Ch 15).  
**Evaluation metrics** — Criteria used to assess the performance of an AI system, such as accuracy, precision, recall, etc. (Ch 16).  

This glossary provides concise definitions and references to the chapters where each term is discussed.
