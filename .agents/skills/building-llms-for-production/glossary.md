# Glossary

Here is an organized glossary of key terms from the provided chapters, presented in alphabetical order with their respective chapter references:

- **Active Learning**: A machine learning technique where models iteratively select the most informative data points for training.

- **Alibi (ALiBi)**: A positional encoding method used in transformers to decompose inter-head biases, enhancing model performance.

- **Anaphora Resolution**: The process of identifying antecedents for anaphoric references in text processing tasks.

- **Application-Specific Language Models (ASLMs)**: Customized LLMs tailored for specific domains or tasks.

- **Autonomous Agents**: AI systems designed to perform tasks without human intervention, often using LLMs for decision-making.

- **Average Precision (AP)**: A metric used in retrieval systems to evaluate the quality of search results based on user feedback.

- **Bash Command**: A shell command for executing system-level operations, such as creating directories or files.

- **Best Time Research**: An application where an LLM is used to determine optimal times for activities like visiting locations.

- **Bigrams**: Two-word sequences used in text processing and language modeling tasks.

- **Binary Classification**: A classification task with two possible outcomes, often used in sentiment analysis.

- **BPE (Byte Pair Encoding)**: A tokenization method that splits text into subword units based on frequency of co-occurrence.

- **Boolean Queries**: Queries using logical operators to combine search terms for precise information retrieval.

- **Boxed Prompts**: A prompt engineering technique where specific instructions are highlighted to guide the model's output.

- **Breadth-First Search (BFS)**: An algorithm used in graph traversal and shortest path finding, applied in various AI tasks.

- **Cache Hits**: The number of times a data request is retrieved directly from memory rather than being fetched from storage.

- **Candidate Generation**: The process of generating potential solutions or options for problem-solving tasks.

- **Causal Language Models**: Models that predict the next word based on preceding words, capturing sequential dependencies.

- **Chain of Thought Prompting**: A prompt engineering method where multiple prompts guide the model through a logical sequence of thoughts.

- **ChatGPT**: An AI tool by OpenAI that generates human-like text responses to various inputs.

- **CheXNLP**: A medical X-ray image analysis model used for diagnosing lung conditions like pneumonia and tuberculosis.

- **Classification**: Assigning items into predefined categories based on their features or characteristics.

- **Clustering**: The process of grouping data points into clusters based on similarity, often used in exploratory data analysis.

- **Code Generation**: The automated creation of code (e.g., Python scripts) for tasks like model deployment and data processing.

- **Command Line Interface (CLI)**: A user interface where commands are entered to perform actions without a graphical interface.

- **Context Window**: The range of previous tokens or information considered when generating the next token in a sequence.

- **Deep Learning**: A subset of machine learning that uses neural networks with multiple layers to learn complex patterns from data.

- **Denoising Diffusion Probabilistic Models (DDPM)**: A generative model that learns to remove noise step by step, used for image generation and other tasks.

- **Dataset Version Control**: Managing different versions of datasets to track changes and ensure reproducibility in experiments.

- **Deep Lake**: A service providing tools and infrastructure for managing and serving embeddings efficiently.

- **Dense Retrieval**: A retrieval strategy where a small number of highly relevant documents are returned for each query.

- **Dense Vector Representation**: Representing data points as dense vectors, which are continuous numerical representations in machine learning models.

- **Differential Privacy**: A method to protect the privacy of individual records in datasets by adding noise during analysis or computation.

- **Embedding Models**: Techniques that map words, phrases, or other text elements into low-dimensional vector spaces for processing.

- **Ensemble Learning**: A method where multiple models are trained and their predictions combined to improve overall performance.

- **Environment Setup**: The process of configuring the necessary software, hardware, and dependencies required for a project or task.

- **Error Fixing**: Techniques used to identify and correct errors in generated text or model outputs.

- **Ethical AI**: Practices and guidelines ensuring that AI systems are developed, deployed, and used responsibly and ethically.

- **Explainable AI (XAI)**: Methods to make the decision-making process of AI models understandable and transparent to users.

- **External Validation**: Testing a model's performance on data outside its training set to evaluate generalization capability.

- **False Positives**: Incorrectly classified instances where an item is incorrectly labeled as positive when it should be negative.

- **File Paths**: The hierarchical structure used to locate files within a computing environment, consisting of directories and files.

- **Fine-Tuning**: Adjusting the parameters of an existing model using new training data to improve its performance on specific tasks.

- **Fragile Knowledge**: Information that is easily disrupted or corrupted by noise in datasets, affecting model reliability.

- **Function Approximation**: The process of modeling relationships between variables to predict outcomes based on input features.

- **Gated Pressure-Insensitive Transistor Quads (GPTQ)**: A type of hardware component used for quantization in machine learning models.

- **GitHub Actions**: A feature that automates workflows and tasks within the GitHub platform, useful for CI/CD pipelines.

- **Global Context**: The overall context or information available to a model when processing input, encompassing prior knowledge and experiences.

- **GPT (Generative Pre-trained Transformer)**: An advanced LLM developed by OpenAI that can generate coherent and contextually relevant text.

- **GPTQ (Gated Pressure-Insensitive Transistor Quads)**: A hardware component used for quantization in machine learning models, often applied to reduce model size or improve efficiency.

- **Graph Neural Networks (GNNs)**: A type of neural network designed to operate on graph data structures, capturing relationships between nodes and edges.

- **Grid Search**: A method for tuning hyperparameters by testing predefined parameter combinations exhaustively across specified values.

- **Grouped Prompts**: Multiple prompts presented together to guide the model's output in a structured manner.

- **Hugging Face**: An open-source platform providing tools, libraries, and resources for building and training machine learning models, particularly NLP-focused.

- **Hyperparameters**: Configuration settings used in machine learning models that are not learned from the data but set before training begins.

- **Instance Segmentation**: A computer vision task where each pixel in an image is labeled to identify and segment specific objects within it.

- **Knowledge Distillation**: The process of transferring knowledge from a large model (teacher) to a smaller model (student), often for deployment purposes.

- **Label Smoothing**: A regularization technique used during training to prevent the model from overfitting by making labels less certain.

- **Language Model Fine-Tuning**: Adjusting an existing language model using new data or tasks to improve its performance on specific applications.

- **Llama (LLaMA)**: An open-source large language model based on LLaNet, known for its high performance and scalability.

- **Lightning Speed**: The ability of a system or model to process information quickly, often used in real-time applications.

- **Logits**: Unnormalized prediction scores from the output layer of a neural network, which are converted into probabilities using functions like softmax.

- **Long-Tail Distribution**: A statistical distribution where most observations have small values, but a few have very large values, affecting model performance on rare cases.

- **Masked Language Modeling (MLM)**: A pre-training task for language models where parts of the input text are masked to encourage prediction of missing words.

- **Maximum Entropy Markov Model (MEMM)**: A classification model that uses maximum entropy principles to predict sequences, often used in NLP tasks like named entity recognition.

- **Mean Average Precision (MAP)**: A metric for evaluating retrieval systems based on the average precision across all queries.

- **Microservices**: Smaller, independent components of a larger system, each handling specific business functionalities or processes.

- **Missing Values**: Gaps or undefined data points in datasets that can affect model training and performance if not handled properly.

- **Model Deployment**: The process of making an AI model available for use outside the development environment, often involving infrastructure setup and monitoring.

- **Multi-Metric Evaluation**: Using multiple metrics to assess different aspects of a model's performance during evaluation.

- **NLP (Natural Language Processing)**: The field focused on how computers understand, analyze, and generate human language text.

- **Noisy Channel Model**: A communication theory-based approach for modeling errors in data transmission or processing.

- **Normalization**: The process of scaling numeric values to a common range or standard deviation to improve model training and performance.

- **OpenAI (OAG)**: An American company that develops AI technologies, particularly known for its language models like GPT.

- **Optimal Policy**: A decision-making strategy in reinforcement learning where actions are chosen based on maximizing cumulative rewards.

- **Output Parsers**: Tools or algorithms used to format model outputs into a specific structure or language suitable for human consumption.

- **Parameter Sharing**: A technique in neural networks where parameters (weights) are reused across different parts of the network, reducing the number of parameters and computational resources needed.

- **Part-of-Speech Tagging (POS)**: The process of labeling words in a text with their corresponding grammatical parts of speech to aid in parsing and understanding sentences.

- **Path Regularization**: A technique used in machine learning to prevent overfitting by adding a penalty term based on the complexity of the model's parameters.

- **Prompt Engineering**: The practice of designing prompts (input instructions) to guide AI models towards desired responses, often involving creativity and domain-specific knowledge.

- **Python Programming**: A high-level programming language known for its simplicity and versatility, widely used in data analysis, web development, and AI applications.

- **Query Processing**: The task of understanding and executing user queries, often involving natural language processing and information retrieval techniques.

- **Reinforcement Learning (RL)**: A type of machine learning where an agent learns to make decisions by performing actions to maximize cumulative rewards.

- **Role Prompting**: Using prompts that specify the role or perspective from which a query should be answered, enhancing the model's ability to provide contextually relevant responses.

- **Run-Length Encoding (RLE)**: A method for compressing binary data sequences by encoding consecutive repeated values into a single value and its count.

- **Self-Attention**: A mechanism in neural networks that allows models to consider all parts of an input sequence when processing each element, enabling the model to capture long-range dependencies.

- **Squad (SuperGLUE)**: A benchmark dataset for reading comprehension tasks, consisting of multiple datasets with diverse language understanding requirements.

- **Summarization**: The process of condensing a larger text into a shorter version while retaining its main points and key information.

- **Tokenization**: The division of text into tokens or pieces, often based on spaces, punctuation, or predefined rules, to facilitate processing by AI models.

- **Transfer Learning**: A machine learning technique where knowledge gained from training one model is applied to another related task, often with minimal retraining.

- **Tuning Hyperparameters**: Adjusting the parameters of a machine learning model during the training process to optimize its performance on unseen data.

- **Unsupervised Learning**: A type of machine learning that involves training models on unlabeled data to find hidden patterns or intrinsic structures within the data.

- **Vector Indexing**: The process of creating an index structure for efficient retrieval of vectors in high-dimensional spaces, often used in nearest neighbor search algorithms.

- **Vocabulary Expansion**: Increasing the size of a model's vocabulary by adding new words or tokens that were previously not present in the training data.

- **Visible Learning**: A concept from education and cognitive science emphasizing the importance of making learning visible through observable actions, interactions, and outcomes.

- **Vision-Language Models (VLMs)**: AI models designed to understand and process both visual and textual information, often used for tasks like image captioning or visual question answering.

- **Web Search**: The process of retrieving information from the web using search engines or other tools, often involving ranking results based on relevance criteria.

- **Zero-Shot Prompting**: A prompt engineering technique where a model is instructed to generate responses without prior examples or context for unseen tasks.

This glossary provides clear definitions and references for each key term, aiding in understanding their roles and applications within the chapter titles.
